# 03. 主 Notebook 规范

## 1. Notebook 定位

`notebooks/main_analysis.ipynb` 是项目的总控执行文件，也是分析叙事文档。

它必须承担以下职责：

- 读取并确认 `docs/analysis_framework.md`。
- 读取并确认 `reports/final/final_report_structure.md`。
- 说明本次分析基于的业务背景、决策目标、核心业务问题和分析边界。
- 加载环境、配置和依赖。
- 连接数据库。
- 调用 Python 脚本执行 SQL 文件。
- 加载和校验数据。
- 调用 Python 模块完成清洗、特征、分析、建模和可视化。
- 展示关键中间结果。
- 导出图表、表格和报告输入素材。
- 支撑脚本生成结果呈现型最终报告。

Notebook 不应承担大量底层函数定义。复杂逻辑必须下沉到 `src/` 目录中的 Python 脚本。

## 2. Notebook 必须包含的章节

主 Notebook 必须按以下顺序组织：

```text
1. 分析框架确认
2. 环境、依赖与配置加载
3. 数据库连接与 SQL 执行说明
4. 数据源与字段理解
5. 数据加载
6. 数据质量检查
7. 数据清洗与转换
8. 探索性数据分析
9. 业务问题分析
10. 统计分析，如适用
11. 机器学习建模，如适用
12. 结果验证与稳健性检查
13. 图表、表格和报告输入素材导出
14. 最终报告脚本生成
15. 项目完成检查清单
```

如果某一章节不适用于当前项目，必须在 Notebook 中说明“不适用”的原因。

## 3. Notebook 备注规范

每一个主要步骤之前必须有 Markdown 说明。禁止只写代码而不解释目的。

每个主要步骤的 Markdown 说明应该包含：

```markdown
## 步骤 XX：步骤名称

### 目的
说明本步骤要解决什么问题。

### 输入
说明使用哪些数据表、SQL 文件、配置文件、数据文件或中间对象。

### 处理逻辑
说明主要处理方法、计算逻辑、筛选条件、建模方法或检验方法。

### 输出
说明本步骤会生成哪些 DataFrame、图表、结果表、中间文件或报告素材。

### 注意事项
说明数据口径、异常情况、风险、限制或需要人工复核的内容。
```

关键代码 cell 上方必须有简短说明，尤其是以下内容：

- 数据库连接。
- SQL 脚本执行。
- 数据清洗规则。
- 指标计算逻辑。
- 统计检验方法。
- 特征工程规则。
- 模型训练和评估逻辑。
- 图表导出逻辑。
- 最终结果保存逻辑。

## 4. Notebook 执行规范

Notebook 必须满足：

- 可以从第一个 cell 到最后一个 cell 顺序执行。
- 不依赖隐藏变量、历史执行状态或手动修改内存对象。
- 不保留无关实验代码、失败代码和临时调试输出。
- 不在 Notebook 中硬编码数据库密码、密钥、个人路径或敏感参数。
- 所有随机过程必须设置随机种子。
- 所有关键输出必须保存为文件。

建议在 Notebook 开头设置以下项目参数：

- 项目名称。
- 分析日期。
- 数据时间范围。
- 运行环境。
- 随机种子。
- 输出目录。
- 是否连接生产数据库。
- 是否只读执行 SQL。

## 5. Notebook 代码组织原则

Notebook 应负责组织流程和呈现结果，不应堆积大量函数定义。

适合留在 Notebook 中的内容：

- 项目背景和分析目标。
- 参数设置和配置读取。
- 调用 `src/` 函数的流程代码。
- 展示关键中间结果。
- 对图表、统计结果、模型结果的解释。
- 导出报告输入素材。

应该下沉到 `src/` 的内容：

- 数据库连接。
- SQL 文件读取和执行。
- 数据质量检查函数。
- 清洗和转换函数。
- 指标计算函数。
- 特征工程函数。
- 统计检验函数。
- 模型训练和评估函数。
- 通用绘图函数。
- 结果导出函数。

## 6. 主 Notebook 推荐骨架

`notebooks/main_analysis.ipynb` 应该遵循以下骨架：

```markdown
# 项目名称

## 1. 分析框架确认

### 目的
确认本次分析基于 docs/analysis_framework.md 中的业务分析框架。

### 输出
明确决策目标、核心业务问题、业务拆解框架、分析路径和分析边界。

## 2. 环境、依赖与配置加载

### 目的
加载必要依赖、项目配置、路径和随机种子。

## 3. 数据库连接与 SQL 执行说明

### 目的
通过 Python 连接数据库，并执行项目所需 SQL 文件。

### 注意事项
说明是否只读、是否涉及生产库、是否使用敏感数据。

## 4. 数据源与字段理解

### 目的
说明数据表、字段、粒度、时间范围和核心口径。

## 5. 数据加载

### 目的
加载 SQL 查询结果或本地数据文件。

## 6. 数据质量检查

### 目的
检查缺失、重复、异常、主键、时间范围和关键指标。

## 7. 数据清洗与转换

### 目的
执行可复现的数据清洗、字段转换和宽表构建。

## 8. 探索性数据分析

### 目的
理解分布、趋势、差异和潜在假设。

## 9. 业务问题分析

### 目的
围绕 docs/analysis_framework.md 中确认的业务问题逐项分析。

## 10. 统计分析，如适用

### 目的
使用统计方法验证假设或量化不确定性。

## 11. 机器学习建模，如适用

### 目的
在确有预测、分类、聚类或排序需求时完成建模和评估。

## 12. 结果验证与稳健性检查

### 目的
检查结论是否稳定、是否受异常值或样本选择影响。

## 13. 图表、表格和报告输入素材导出

### 目的
将关键图表、结果表和 reports/final/report_inputs.yaml 保存到 reports/。

## 14. 最终报告脚本生成

### 目的
运行 scripts/generate_final_report.py 输出结果呈现型最终报告。

### 要求
默认报告只呈现结果、图表、表格、口径和输出文件路径，不自动生成洞察或业务解读。

### 输出
reports/final/final_analysis_report.md

## 15. 项目完成检查清单
```

## 7. 交付前 Notebook 清理要求

交付前必须清理：

- 无关实验代码。
- 失败代码。
- 临时打印。
- 只服务调试的中间输出。
- 未使用变量和未使用导入。
- 与项目无关的图表。

交付前必须确认：

- 所有 cell 已按顺序运行。
- 没有报错。
- 输出文件已生成。
- 图表和结果表路径正确。
- reports/final/report_inputs.yaml 已生成。
- 最终报告由 scripts/generate_final_report.py 生成。
