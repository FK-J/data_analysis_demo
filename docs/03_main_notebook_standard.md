# 03. 主 Notebook 规范

## 1. Notebook 定位

`notebooks/main_analysis.ipynb` 是项目的总控执行文件，也是分析叙事文档和交互式分析工作台。

它必须承担以下职责：

- 读取并确认 `docs/analysis_framework.md`。
- 读取并确认 `reports/final/final_report_structure.md`。
- 说明本次分析基于的业务背景、决策目标、核心业务问题和分析边界。
- 加载环境、配置和依赖。
- 连接数据库。
- 调用 Python 脚本执行 SQL 文件。
- 加载和校验数据。
- 调用 Python 模块完成清洗、特征、分析、建模和可视化。
- 展示关键中间结果。
- 导出图表、表格和报告输入素材。
- 支撑脚本生成结果呈现型最终报告。

Notebook 不应承担大量底层函数定义。复杂逻辑必须下沉到 `src/` 目录中的 Python 脚本。

数据分析通常不是一次从头到尾完成的线性工作。Notebook 必须支持分析师围绕某个中间步骤局部调试、调整逻辑并继续运行；但交付前必须从头到尾顺序执行一次，确认最终结果可复现。

## 2. Notebook 必须包含的章节

主 Notebook 必须按以下顺序组织：

```text
1. 分析框架确认
2. 环境、依赖与配置加载
3. 数据库连接与 SQL 执行说明
4. 数据源与字段理解
5. 数据加载
6. 数据质量检查
7. 数据清洗与转换
8. 探索性数据分析
9. 业务问题分析
10. 统计分析，如适用
11. 机器学习建模，如适用
12. 结果验证与稳健性检查
13. 图表、表格和报告输入素材导出
14. 最终报告脚本生成
15. 项目完成检查清单
```

如果某一章节不适用于当前项目，必须在 Notebook 中说明“不适用”的原因。

## 3. Notebook 备注规范

每一个主要步骤之前必须有 Markdown 说明。禁止只写代码而不解释目的。

每个大章节只保留一个标题，例如：

```markdown
## 步骤 XX：步骤名称
```

标题下面的备注内容统一使用正文，不再使用 `### 目的`、`### 输入`、`### 输出` 这类多层小标题。如果确实需要强调某一项，使用加粗正文标签即可，例如：

```markdown
## 步骤 XX：步骤名称

**目的：** 说明本步骤要解决什么问题。

**上游依赖：** 说明运行本节前必须完成哪些章节、确认哪些口径或准备哪些对象。

**输入文件/对象：** 说明使用哪些数据表、SQL 文件、配置文件、数据文件、DataFrame 或中间对象。

**输出文件/对象：** 说明本步骤会生成哪些 DataFrame、图表、结果表、中间文件或报告素材。

**是否可缓存：** 说明本节中间结果是否可以落地复用，以及哪些变更会导致缓存失效。

**调试提示：** 说明人工调试本节时常见的回退章节、风险、限制或需要复核的内容。
```

关键代码 cell 上方必须有简短说明，尤其是以下内容：

- 数据库连接。
- SQL 脚本执行。
- 数据清洗规则。
- 指标计算逻辑。
- 统计检验方法。
- 特征工程规则。
- 模型训练和评估逻辑。
- 图表导出逻辑。
- 最终结果保存逻辑。

如果某一节的逻辑会频繁迭代，应该在备注中说明它的缓存位置、重跑条件和下游影响范围，方便分析师局部调试，而不是每次完整执行整个 Notebook。

## 4. Notebook 执行规范

Notebook 必须满足：

- 可以从第一个 cell 到最后一个 cell 顺序执行。
- 日常分析允许局部运行，但交付前必须完整顺序运行。
- 局部运行时应依赖明确的输入文件、缓存文件或上游对象，不应依赖无法追溯的隐藏变量。
- 不依赖历史执行状态或手动修改内存对象生成最终交付结果。
- 不保留无关实验代码、失败代码和临时调试输出。
- 不在 Notebook 中硬编码数据库密码、密钥、个人路径或敏感参数。
- 所有随机过程必须设置随机种子。
- 所有关键输出必须保存为文件。

建议在 Notebook 开头设置以下项目参数：

- 项目名称。
- 分析日期。
- 数据时间范围。
- 运行环境。
- 随机种子。
- 输出目录。
- 是否连接生产数据库。
- 是否只读执行 SQL。

## 5. Notebook 代码组织原则

Notebook 应负责组织流程和呈现结果，不应堆积大量函数定义。

适合留在 Notebook 中的内容：

- 项目背景和分析目标。
- 参数设置和配置读取。
- 调用 `src/` 函数的流程代码。
- 展示关键中间结果。
- 对图表、统计结果、模型结果的解释。
- 导出报告输入素材。

应该下沉到 `src/` 的内容：

- 数据库连接。
- SQL 文件读取和执行。
- 数据质量检查函数。
- 清洗和转换函数。
- 指标计算函数。
- 特征工程函数。
- 统计检验函数。
- 模型训练和评估函数。
- 通用绘图函数。
- 结果导出函数。

## 6. 主 Notebook 推荐骨架

`notebooks/main_analysis.ipynb` 应该遵循以下骨架：

```markdown
# 项目名称

## 1. 分析框架确认

**目的：** 确认本次分析基于 docs/analysis_framework.md 中的业务分析框架。

**上游依赖：** 用户已确认业务背景、决策目标、核心业务问题和分析边界。

**输入文件/对象：** docs/analysis_framework.md。

**输出文件/对象：** 分析框架确认状态；必要时更新 docs/analysis_framework.md。

**是否可缓存：** 不缓存。业务问题或分析边界变化时必须重新确认。

**调试提示：** 后续结果显示分析路径不合理时，先回到本节更新框架。

## 2. 环境、依赖与配置加载

**目的：** 加载必要依赖、项目配置、路径和随机种子。

**上游依赖：** 当前运行目录位于项目根目录或 notebooks/。

**输入文件/对象：** configs/analysis_config.yaml。

**输出文件/对象：** 项目路径、配置对象、随机种子和 import 路径。

**是否可缓存：** 不建议跨会话缓存。重启 Kernel 后先运行本节。

**调试提示：** 路径、配置或模块导入异常时优先重跑本节。

## 3. 数据库连接与 SQL 执行说明

**目的：** 通过 Python 连接数据库，并执行项目所需 SQL 文件。

**上游依赖：** 配置加载完成；数据库配置、SQL 文件和查询参数已准备。

**输入文件/对象：** configs/database.yaml、sql/ 目录、src/db/ 模块。

**输出文件/对象：** 数据库连接对象、查询结果 DataFrame 和 SQL 执行元数据。

**是否可缓存：** 可缓存稳定查询结果到 data/interim/；SQL、参数、数据源或时间范围变化时失效。

**调试提示：** 默认只读执行。涉及写库、删表、更新、建表或覆盖表时必须先确认风险。

## 4. 数据源与字段理解

**目的：** 说明数据表、字段、粒度、时间范围和核心口径。

**上游依赖：** 分析框架已确认；数据源可访问。

**输入文件/对象：** 数据库表结构、SQL 查询元数据、本地数据文件或业务数据字典。

**输出文件/对象：** 数据源说明、字段理解记录；可选 reports/tables/data_dictionary.csv。

**是否可缓存：** 可缓存字段字典和数据画像；schema、字段含义或时间范围变化时失效。

**调试提示：** 字段理解变更通常影响后续全部数据处理和分析。

## 5. 数据加载

**目的：** 加载 SQL 查询结果或本地数据文件。

**上游依赖：** 数据源与字段理解已完成；数据库连接或文件路径已确认。

**输入文件/对象：** SQL 查询结果、data/raw/、data/external/ 和路径配置。

**输出文件/对象：** 原始 DataFrame 或中间 DataFrame；可选 data/interim/ 抽取快照。

**是否可缓存：** 可缓存。源数据、SQL、时间范围、筛选条件或字段选择变化时失效。

**调试提示：** 迭代分析可优先读缓存，交付前需确认缓存版本和参数有效。

## 6. 数据质量检查

**目的：** 检查缺失、重复、异常、主键、时间范围和关键指标。

**上游依赖：** 数据加载完成；主键、时间字段、粒度和关键字段已明确。

**输入文件/对象：** 原始 DataFrame、字段口径、唯一键和校验规则。

**输出文件/对象：** 数据质量检查结果；建议 reports/tables/data_quality_summary.csv。

**是否可缓存：** 可缓存。输入数据或校验规则变化时失效。

**调试提示：** 质量问题可能要求回到数据理解、数据加载或清洗规则章节。

## 7. 数据清洗与转换

**目的：** 执行可复现的数据清洗、字段转换和宽表构建。

**上游依赖：** 数据质量检查完成；清洗规则、异常处理和指标口径已明确。

**输入文件/对象：** 原始或中间 DataFrame、质量检查结果、src/cleaning/ 和 src/features/。

**输出文件/对象：** 清洗后的 DataFrame、特征表或分析宽表；建议 data/interim/ 或 data/processed/。

**是否可缓存：** 强烈建议缓存。原始数据、清洗规则、join 逻辑或指标口径变化时失效。

**调试提示：** 修改本节后，通常需要重跑 EDA、业务分析、导出和报告生成。

## 8. 探索性数据分析

**目的：** 理解分布、趋势、差异和潜在假设。

**上游依赖：** 清洗后数据或分析宽表已生成。

**输入文件/对象：** data/processed/、清洗后的 DataFrame、业务问题和核心分组维度。

**输出文件/对象：** 探索性图表、临时汇总表和中间发现；必要时保存到 reports/。

**是否可缓存：** 可缓存确认有复用价值的图表和汇总表；数据、分组或时间窗口变化时失效。

**调试提示：** 探索阶段允许试错，但应区分临时发现和进入最终交付的结果。

## 9. 业务问题分析

**目的：** 围绕 docs/analysis_framework.md 中确认的业务问题逐项分析。

**上游依赖：** 分析框架、指标口径、清洗后数据和必要 EDA 发现已确认。

**输入文件/对象：** 分析数据、业务分析框架、核心指标口径和业务分组维度。

**输出文件/对象：** 每个业务问题对应的结果表、图表、关键结果和报告素材。

**是否可缓存：** 可按业务问题缓存。输入数据、口径、分组或假设变化时对应缓存失效。

**调试提示：** 如果结果反向影响业务拆解，应回到分析框架和报告结构章节同步调整。

## 10. 统计分析，如适用

**目的：** 使用统计方法验证假设或量化不确定性。

**上游依赖：** 业务问题、统计假设、样本定义和方法选择依据已明确。

**输入文件/对象：** 分析数据、统计参数、分组样本和前提检查结果。

**输出文件/对象：** 统计检验结果、效应量、置信区间和诊断结果。

**是否可缓存：** 可缓存同一样本、方法和参数下的结果；样本、口径或方法变化时失效。

**调试提示：** 不适用时直接用正文说明原因。统计显著性和业务显著性必须分开说明。

## 11. 机器学习建模，如适用

**目的：** 在确有预测、分类、聚类或排序需求时完成建模和评估。

**上游依赖：** 建模场景、标签、特征窗口、样本切分、评估指标和误判成本已确认。

**输入文件/对象：** 建模样本、标签、特征表、随机种子和 src/modeling/。

**输出文件/对象：** 模型评估结果、特征解释、适用边界和必要中间产物。

**是否可缓存：** 可缓存特征矩阵、数据切分和评估结果；标签、特征、切分或参数变化时失效。

**调试提示：** 不适用时直接说明原因。建模调试重点检查数据泄漏和样本穿越。

## 12. 结果验证与稳健性检查

**目的：** 检查结论是否稳定、是否受异常值、时间窗口、样本选择或口径变化影响。

**上游依赖：** 核心业务分析、统计分析或建模结果已经形成。

**输入文件/对象：** 核心结论、关键指标、替代时间窗口、替代样本和替代口径配置。

**输出文件/对象：** 稳健性检查表、敏感性分析图表和结论稳定性说明。

**是否可缓存：** 可缓存单项检查结果；上游数据、口径或核心结论变化时失效。

**调试提示：** 高度敏感的结论应回到上游章节检查，并在最终报告说明局限性。

## 13. 图表、表格和报告输入素材导出

**目的：** 将关键图表、结果表和 reports/final/report_inputs.yaml 保存到 reports/。

**上游依赖：** 业务分析、可选统计/建模、稳健性检查和最终报告结构已确认。

**输入文件/对象：** 关键结果、图表对象、结果 DataFrame、最终报告结构和报告输入模板。

**输出文件/对象：** reports/figures/、reports/tables/ 和 reports/final/report_inputs.yaml。

**是否可缓存：** 不作为分析缓存。每次准备交付候选版本时都应重新导出。

**调试提示：** 检查输出文件是否最新、路径是否存在、是否仍有占位内容或敏感明细。

## 14. 最终报告脚本生成

**目的：** 运行 scripts/generate_final_report.py 输出结果呈现型最终报告。

**上游依赖：** 图表、表格、report_inputs.yaml 和 final_report_structure.md 已准备。

**输入文件/对象：** reports/final/report_inputs.yaml、reports/final/final_report_structure.md 和 scripts/generate_final_report.py。

**输出文件/对象：** reports/final/final_analysis_report.md。

**是否可缓存：** 不缓存。报告输入、结构、图表、表格或洞察内容变化后应重新生成。

**调试提示：** 默认报告只呈现结果，不自动生成洞察或业务解读；如需洞察，必须由用户明确要求并传入 --with-insights。

## 15. 项目完成检查清单
```

## 7. 交付前 Notebook 清理要求

交付前必须清理：

- 无关实验代码。
- 失败代码。
- 临时打印。
- 只服务调试的中间输出。
- 未使用变量和未使用导入。
- 与项目无关的图表。

交付前必须确认：

- 所有 cell 已按顺序运行。
- 没有报错。
- 输出文件已生成。
- 图表和结果表路径正确。
- reports/final/report_inputs.yaml 已生成。
- 最终报告由 scripts/generate_final_report.py 生成。
