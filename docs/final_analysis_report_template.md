# 结果呈现型数据分析报告

本文档是脚本生成最终报告时应遵循的结构说明。具体项目交付时，报告由脚本生成到：

```text
reports/final/final_analysis_report.md
```

默认报告只呈现结果、图表、表格、口径、限制和输出文件路径，不自动生成洞察、归因、业务建议或扩写解读。

如需加入大模型生成的洞察，必须由用户明确要求，并在报告生成脚本中显式开启。

## 1. 汇总结果

本节放在报告最前面，集中呈现最重要的结果。

应包含：

- 关键结果摘要。
- 核心指标总览。
- 关键图表。
- 关键结果表。
- 需要优先关注的问题。

## 2. 分业务问题呈现结果

后续章节按 `docs/analysis_framework.md` 中确认的业务问题组织。

每个业务问题章节应包含：

- 本节目标。
- 具体分析逻辑。
- 关键结果。
- 图表。
- 结果表。
- 口径与限制。

示例：

```markdown
## 2. Q1：业务问题标题

### 本节目标

### 具体分析逻辑

### 关键结果

### 图表

### 结果表

### 口径与限制
```

## 3. 可选洞察

默认不生成本节。

只有用户明确要求，并运行报告生成脚本时传入 `--with-insights`，才可以把输入文件中的洞察内容写入报告。

注意：

- 默认脚本不会调用大模型。
- 默认脚本不会自动生成业务解读。
- 如果需要大模型洞察，应先单独生成并写入 `reports/final/report_inputs.yaml` 的 `insights.items`。

## 4. 附录

附录用于保存复现和理解结果所需的支撑信息：

- 数据范围。
- 指标口径。
- 关键筛选条件。
- 输出文件。
- Notebook、SQL 和 Python 模块路径。
