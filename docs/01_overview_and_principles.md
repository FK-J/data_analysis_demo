# 01. 项目概览与总体原则

## 1. 文档目的

本文档定义数据分析项目的适用范围、规范用语、总体工作原则和 Agent 行为原则。

任何新项目必须先明确：分析不是为了产出代码或图表，而是为了支持业务理解、判断和决策。

## 2. 适用范围

本规范适用于以下类型的数据分析工作：

- 业务分析：指标拆解、用户分析、经营分析、转化分析、留存分析、分群分析等。
- 统计分析：描述性统计、假设检验、相关分析、回归分析、实验分析等。
- 机器学习建模：特征工程、监督学习、无监督学习、模型评估、模型解释等。

项目允许使用 Python 脚本封装可复用逻辑，允许使用 SQL 文件沉淀数据抽取、转换和校验逻辑，但最终必须通过单一 Jupyter Notebook 文件完整执行并复现全部分析流程。

## 3. 规范用语

- 必须：强制要求，除非用户明确说明，否则不得省略。
- 禁止：不得执行或不得出现在最终项目中。
- 应该：默认推荐做法，如不采用，必须在项目说明中解释原因。
- 可以：可选做法，根据项目复杂度决定是否采用。

## 4. 总体工作原则

### 4.1 问题先于数据

在读取、清洗、建模或可视化之前，必须先明确业务问题、分析目标、目标受众和最终决策场景。

必须回答以下问题：

- 本项目要解决什么问题？
- 这个问题服务于什么业务决策？
- 主要使用者是谁？
- 项目成功的判断标准是什么？
- 分析结果需要输出为报告、图表、模型、数据表，还是决策建议？

### 4.2 可复现优先

项目中的关键结论必须能够被复现。

必须保证：

- `notebooks/main_analysis.ipynb` 可以从上到下顺序执行。
- Notebook 不依赖未记录的手动操作。
- 所有核心数据处理逻辑可以追溯到 SQL 文件、Python 脚本或 Notebook cell。
- 所有最终图表、结果表和报告文件均落地保存到 `reports/` 目录。

### 4.3 口径明确

所有核心指标必须有明确口径。

必须说明：

- 指标名称。
- 计算公式。
- 数据来源。
- 时间范围。
- 统计粒度。
- 过滤条件。
- 是否去重。
- 是否包含异常值、取消订单、退款、测试用户等特殊数据。

禁止在未说明口径的情况下输出关键指标结论。

### 4.4 事实、推断与建议分离

分析报告和 Notebook 结论中必须区分：

- 事实：直接由数据计算得到的结果。
- 推断：基于事实进行的解释或判断。
- 建议：面向业务行动的方案。
- 局限性：数据、方法、样本或假设存在的限制。

禁止把相关性直接表述为因果性，除非项目设计和方法能够支持因果推断。

### 4.5 数据安全优先

如果项目涉及个人信息、交易信息、联系方式、精确地址、设备标识、账号标识或其他敏感字段，必须优先考虑数据最小化、脱敏、聚合和访问控制。

禁止在 Notebook、日志、报告、图表或提交文件中暴露敏感明细数据。

### 4.6 方法服务问题

业务分析、统计分析和机器学习建模都只是解决问题的手段。

必须先判断项目问题适合哪类方法，再选择工具。

禁止为了展示技术而强行使用复杂模型。

## 5. Agent 行为原则

### 5.1 开始项目前

必须先明确：

- 用户的问题和目标。
- 数据来源和可访问方式。
- 是否需要连接数据库。
- 是否需要执行 SQL。
- 是否涉及敏感数据。
- 预期交付形式。

如信息不足，应该先提出关键问题，不应直接开始建模或画图。

### 5.2 项目执行中

必须：

- 保持 `notebooks/main_analysis.ipynb` 为主执行入口。
- 将复杂逻辑封装到 `src/`。
- 将 SQL 逻辑保存到 `sql/`。
- 在 Notebook 中解释每一步。
- 对关键中间结果进行检查。
- 及时记录口径、假设和异常。

禁止：

- 只在临时脚本中完成分析，Notebook 无法复现。
- 静默修改原始数据。
- 跳过数据质量检查。
- 直接输出未经验证的结论。
- 在不适合建模的问题上强行使用机器学习。

### 5.3 项目完成前

必须检查：

- Notebook 是否可以顺序运行。
- SQL 是否可追溯。
- Python 脚本是否被 Notebook 调用。
- 结果文件是否落地保存。
- 结论是否有数据支撑。
- 局限性是否说明。
- 敏感信息是否泄露。

## 6. 默认执行顺序

后续项目建议按以下顺序推进：

```text
1. 明确业务问题和交付目标
2. 建立项目目录
3. 创建 README.md、.env.example 和配置文件
4. 编写数据库连接和 SQL 执行模块
5. 编写 SQL 抽取与校验脚本
6. 创建 notebooks/main_analysis.ipynb
7. 在 Notebook 中连接数据库并执行 SQL
8. 完成数据质量检查
9. 完成数据清洗和转换
10. 完成探索性分析
11. 完成业务分析
12. 根据需要完成统计分析
13. 根据需要完成机器学习建模
14. 验证结果稳定性
15. 导出图表、表格和最终报告材料
16. 编写最终结论、建议和局限性
17. 根据检查清单完成交付前检查
```
