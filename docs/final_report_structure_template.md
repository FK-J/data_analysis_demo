# 最终报告结构

本文档用于在正式分析前确认最终报告的结果呈现结构。具体项目启动后，建议复制为：

```text
reports/final/final_report_structure.md
```

本文件回答“最终报告会长什么样、每一部分呈现什么结果、对应什么分析逻辑”。它不替代 `docs/analysis_framework.md`，而是把业务分析框架转化为最终报告结构。

## 1. 汇总结果

### 本节目标

在报告最前面集中呈现本次分析最重要的结果，让读者先看到结论性结果，再进入分问题展开。

### 具体分析逻辑

- 汇总各核心业务问题的关键结果。
- 比较不同问题之间的重要性、影响范围和优先级。
- 提炼最需要业务方关注的变化、差异、异常或机会。
